Analyzing Forensic Features in AI Images

A deep dive into how CNN models can utilize Error Level Analysis (ELA) and Fast Fourier Transform (FFT) to distinguish between real and AI-generated media.

When I was trying to figure out a title for my Final Year Project (FYP), I was totally stuck. But one day, while scrolling through social media, I realized something pretty scary: AI-generated images and videos are getting way too realistic. Nowadays, it's so hard to tell what's real and what's fake with just our bare eyes.

That got me curious, so I started digging through Google Scholar to see how researchers are actually detecting AI content. I found a really interesting paper that suggested using multiple forensic features at the same time. While there are already methods out there using things like Error Level Analysis (ELA) or Photo Response Non-Uniformity (PRNU), a lot of them share the same issue: they struggle with "cross-generation." Basically, a model might be great at catching Midjourney images but completely fail when tested on Stable Diffusion.

To build a more robust system for my FYP, I decided to focus on this gap. At first, I wanted to train a system using tons of different generative models. But to keep the project realistic for my timeframe, I decided to scale it down. I chose to replicate an existing baseline study so I could directly compare my results with theirs.

The main difference in my approach is that I built my system using PyTorch. I chose PyTorch simply because it made it way easier for me to modify the very first layer of the CNN model to accept my custom forensic features.

The Forensic Features Setup

Speaking of forensic features, here is a quick breakdown of what I actually used to catch the AI:

  • troubleshoot
    Fast Fourier Transform (FFT): This helps me see the high-frequency artifacts in an image. By converting normal spatial data into frequency data, the unnatural pixel patterns left behind by AI generators become super obvious.
  • compress
    Error Level Analysis (ELA): This feature highlights compression differences. Since an AI generates an image as one single flat layer, applying ELA will show a very uniform compression. A real photo that has been edited, on the other hand, will show clear differences where the image was manipulated.
  • blur_on
    Laplace High-Pass Filtering: I also added this filter to help the model learn the noise residuals. It basically extracts the hidden texture or hidden noise patterns that the AI accidentally leaves behind.
